# QVAC TTS ONNX Addon v0.5.0 Release Notes

This release adds the Supertonic TTS engine as an alternative to Chatterbox. You can choose between the two engines by configuration: the addon auto-detects Supertonic when you pass `modelDir` and `voiceName`, or when you pass Supertonic-specific paths. Windows support for Supertonic and several correctness and test improvements are included.

## Features

### Supertonic TTS Engine

The addon now supports a second TTS engine, Supertonic, alongside Chatterbox. Engine selection is automatic based on the arguments you provide. If you pass `modelDir` and `voiceName`, or any of the Supertonic-specific paths (`textEncoderPath`, etc.), the addon uses the Supertonic engine; otherwise it uses Chatterbox.

You can use Supertonic with a single model directory and voice name:

```js
const model = new ONNXTTS({
  modelDir: path.join(__dirname, 'models', 'supertonic'),
  voiceName: 'F1',
  speed: 1,
  numInferenceSteps: 5
}, { language: 'es' })
await model.load()
const response = await model.run({ input: 'Hola mundo!', type: 'text' })
```

Alternatively, you can pass explicit paths to the tokenizer, text encoder, latent denoiser, voice decoder, and voices directory. TypeScript users get `SupertonicTTSArgs` and `ChatterboxTTSArgs`; the unified `ONNXTTSArgs` type allows either shape. An example script, `examples/example-supertonic-tts.js`, demonstrates Supertonic usage.
